# Zig Mono 构建产物对比

## 文件大小对比

### 静态链接版本
| 文件 | 大小 |
|------|------|
| static-app | 76K (77,624 bytes) |
| **总计** | **76K (77,624 bytes)** |

### 动态链接版本
| 文件 | 大小 |
|------|------|
| dynamic-app | 55K (56,312 bytes) |
| libcommon.dylib | 74K (75,888 bytes) |
| libmathlib.dylib | 75K (76,976 bytes) |
| libstringlib.dylib | 75K (77,232 bytes) |
| **总计** | **280K (286,408 bytes)** |

## 差异
- 动态链接版本比静态链接版本大 **208,784 bytes** (268.97%)
- 静态链接：单个可执行文件
- 动态链接：1个可执行文件 + 3个动态库

## 分析

与Go版本相比，Zig的编译产物有以下特点：

1. **静态链接版本更小**：Zig的静态链接应用只有76K，远小于Go的2.3M
2. **动态链接版本也更小**：Zig的动态链接总大小为280K，远小于Go的7.4M
3. **相对大小比例类似**：动态链接版本约是静态链接版本的3.7倍（Go为3.2倍）

Zig的优势在于：
- 没有运行时（runtime），编译产物更小
- 更直接的C ABI集成，动态库开销小
- 优秀的优化能力，代码体积更紧凑
